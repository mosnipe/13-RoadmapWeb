# コミットメッセージガイドライン

このプロジェクトでは、日本語でコミットメッセージを記述します。

## コミットメッセージの形式

```
<種類>(<スコープ>): <簡潔な説明>

<詳細説明（必要に応じて）>

<フッター（必要に応じて）>
```

## 変更の種類

- **feat**: 新機能の追加
- **fix**: バグ修正
- **docs**: ドキュメントの変更
- **style**: コードフォーマットの変更（動作に影響しない）
- **refactor**: リファクタリング（機能追加・バグ修正以外の変更）
- **perf**: パフォーマンスの改善
- **test**: テストの追加・修正
- **chore**: ビルドプロセスやツールの変更
- **ui**: UI/UXの変更
- **config**: 設定ファイルの変更

## スコープ（オプション）

変更が影響する範囲を指定します。例：

- `integrations`: 統合設定画面
- `roadmap`: ロードマップ表示画面
- `parser`: ファイル解析機能
- `storage`: ストレージ管理
- `ui`: UIコンポーネント
- `config`: 設定ファイル

## 例

### 基本的なコミット
```
feat(integrations): GitHubファイル読み込み機能を追加
```

### 詳細説明付き
```
fix(roadmap): タイムライン表示の日付ソートを修正

日付が正しくソートされていなかった問題を修正。
年月でグループ化する際の比較ロジックを改善。
```

### スコープなし
```
docs: READMEに使用方法を追加
```

### 複数の変更を含む場合
```
feat(ui): 統合設定画面のデザインを更新

- 白基調のデザインに変更
- メインカラーを青に変更
- アクセントカラーにオレンジを追加
- すべてのテキストを日本語化
```

## コミットメッセージテンプレートの設定

プロジェクトルートの `.gitmessage` ファイルがコミットメッセージのテンプレートとして設定されています。

初回設定（プロジェクトで一度だけ実行）:
```bash
git config --local commit.template .gitmessage
```

これにより、`git commit` を実行すると自動的にテンプレートが表示されます。

## AIツール（Cursor/GitHub Copilot）での使用

AIツールが自動生成するコミットメッセージも、このガイドラインに従って日本語で生成されるように設定してください。

Cursorの設定例:
- プロジェクトの設定でコミットメッセージの言語を日本語に設定
- `.cursorrules` ファイルにコミットメッセージのガイドラインを追加
